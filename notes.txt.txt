1. Git เก็บ ข้อมูลไว้ไหน
	Git เก็บข้อมูลไว้ในระบบควบคุมเวอร์ชัน (Version Control System) ซึ่งประกอบด้วย Git repository หรือโครงการ Git โดยข้อมูลทั้งหมดที่เกี่ยวข้องกับโปรเจ็กต์ ซ้ำกัน และประวัติการเปลี่ยนแปลง (commit) จะถูกเก็บไว้ใน Git repository ซึ่งอาจอยู่ในเครื่องคอมพิวเตอร์ของคุณ (local repository) หรือบนเว็บโฮสติ้งแพลตฟอร์มเช่น GitHub, GitLab, Bitbucket ซึ่งเป็นรูปแบบของ remote repository ที่สามารถเชื่อมต่อกับ local repository ของคุณได้
2. อธิบายวิธีการบันทึกการเปลี่ยนแปลงใน Git ?
	การเพิ่มไฟล์ในการ Stage: เมื่อคุณทำการเพิ่มไฟล์ใหม่หรือทำการเปลี่ยนแปลงในไฟล์ที่มีอยู่ คุณต้องเพิ่มไฟล์เหล่านั้นในการ Stage โดยใช้คำสั่ง git add เพื่อบอก Git ว่าคุณต้องการจะเก็บการเปลี่ยนแปลงในไฟล์ดังกล่าวไว้เพื่อทำการ commit
	การทำการ Commit: เมื่อไฟล์ถูก Stage แล้ว คุณสามารถทำการ commit เพื่อบันทึกการเปลี่ยนแปลงที่เกิดขึ้นในไฟล์ การ commit จะทำให้ Git บันทึกประวัติการเปลี่ยนแปลงของไฟล์เหล่านั้นใน Git repository
	การแสดงประวัติการ Commit: คุณสามารถดูประวัติการ commit ทั้งหมดที่เกิดขึ้นใน Git repository ด้วยคำสั่ง git log เพื่อตรวจสอบการเปลี่ยนแปลงที่เกิดขึ้นในโปรเจ็กต์ของคุณ

3. อธิบายวิธีการบันทึกเกี่ยวกับการเปลี่ยนแปรง ทำอย่างไร
	1.เพิ่มไฟล์ในการ Stage (Staging): เมื่อคุณทำการเปลี่ยนแปลงในไฟล์ หรือเพิ่มไฟล์ใหม่ลงในโปรเจ็กต์ คุณต้องเพิ่มไฟล์เหล่านั้นในการ Stage โดยใช้คำสั่ง git add เพื่อบอก Git ว่าคุณต้องการให้ Git จดจำการเปลี่ยนแปลงในไฟล์เหล่านั้นไว้ โดยการ Stage คือการเตรียมไฟล์ให้พร้อมที่จะทำการ commit

	2.ทำการ Commit: เมื่อไฟล์ถูก Stage แล้ว คุณสามารถทำการ commit เพื่อบันทึกการเปลี่ยนแปลงที่เกิดขึ้นในไฟล์ การ commit จะเป็นการทำการบันทึกสถานะปัจจุบันของ Git repository ซึ่งรวมถึงการเปลี่ยนแปลงในไฟล์ที่คุณได้ทำการ Stage ไว้ โดยการ Commit คือการบันทึกการเปลี่ยนแปลงที่เกิดขึ้นในรูปแบบของ "snapshot" ที่เกิดขึ้นในจุดเวลานั้นๆ

	3.การรับรู้ประวัติการ Commit: คุณสามารถดูประวัติการ commit ทั้งหมดที่เกิดขึ้นใน Git repository ด้วยคำสั่ง git log เพื่อตรวจสอบรายการ Commit ที่คุณได้ทำไว้ทั้งหมด คุณจะเห็นรายละเอียดเกี่ยวกับผู้ commit วันที่ และข้อความ Commit ที่คุณได้ระบุในขณะที่ทำการ Commit
